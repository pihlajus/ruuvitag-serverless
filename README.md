# Ruuvitag Serverless

Serverless replacement for an EC2-based home IoT pipeline. Replaces a
single t3.micro running InfluxDB + Grafana with managed AWS services:
IoT Core → DynamoDB → Grafana Cloud.

I run a few Ruuvitag sensors at home for temperature/humidity logging.
The original setup served well for years but was overkill for the
volume — this is the small-volume serverless version.

## Why

- Less operating cost (~$19/month → ~$2-5/month)
- No EC2 to patch or restart
- No public InfluxDB port; X.509 mTLS instead
- Backups handled by managed services

## Architecture

```
Raspberry Pi (at home)
  │ MQTT over mTLS
  ▼
AWS IoT Core
  ├── IoT Rule ─→ DynamoDB (mac partition / ts_ms sort, full archive)
  └── IoT Rule ─→ Lambda ─→ Grafana Cloud Prometheus (last 14 days)
```

Two destinations, one for each thing Grafana wants:

- **DynamoDB** is the long-term archive. On-demand pricing, queryable
  by `mac` + time range, used for history older than 14 days via the
  community DynamoDB datasource plugin.
- **Grafana Cloud Prometheus** is the live dashboard backend. PromQL
  panels are nicer than DynamoDB's query API for time-series, and the
  Free tier's 14-day retention covers day-to-day monitoring.

Originally planned with Timestream as a single store, but Timestream
is not available in `eu-north-1`. The split keeps storage cost in
DynamoDB (~$0.50/month for 6 years of data) and dashboard ergonomics
in Prometheus (free).

## Repository structure

```
.
├── bootstrap/                  State backend (S3 + DynamoDB lock + KMS)
├── modules/
│   ├── iot-ingestion/         IoT Thing, cert, policy
│   ├── timeseries-storage/    DynamoDB table + IAM role for IoT Rule
│   └── metrics-publisher/     Lambda → Grafana Cloud Prometheus push
├── environments/
│   └── home/                  Composition: modules + IoT Topic Rule
└── pi/                        BLE -> MQTT publisher running on the Pi
```

## Prerequisites

- Terraform 1.9+ (see `.terraform-version`)
- AWS CLI configured for `eu-north-1`

## Getting started

1. **Bootstrap state backend** (once per AWS account):
   ```sh
   cd bootstrap
   terraform init
   terraform apply
   # add backend.tf, then:
   terraform init -migrate-state
   ```

2. **Deploy the home environment**:
   ```sh
   cd environments/home
   terraform init
   terraform plan
   terraform apply
   ```

3. **Set up the Pi** (see [`pi/README.md`](pi/README.md)):
   ```sh
   terraform output -raw certificate_pem > cert.pem
   terraform output -raw private_key > private.key
   # Then follow pi/README.md to install the publisher and systemd unit
   ```

## What's NOT in this repo

- **Device cert + private key** — generated by the `iot-ingestion`
  module, written locally, never committed.
- **`terraform.tfvars`** — gitignored; defaults in `variables.tf`
  cover the home setup.
- **AWS credentials** — supply via `aws configure` or env vars.
